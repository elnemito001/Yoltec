# Opciones de Título de Residencia para el Proyecto Yoltec

## 📋 Información del Proyecto
**Nombre del sistema:** Yoltec - Consultorio Médico Escolar  
**Tecnologías:** Laravel (backend), Angular (web), Flutter (móvil), PostgreSQL/SQLite  
**Características principales:**
- Sistema de IA propio (expert system) para análisis de documentos médicos
- Autenticación de dos factores (2FA)
- Gestión de citas médicas y expedientes
- Roles: Alumno/Paciente, Doctor, Administrador
- Validación de diagnósticos por doctores

---

## 🎯 Opciones de Título de Residencia

### Opción 1 - Técnica/Específica
**"Desarrollo de un Sistema Experto de Inteligencia Artificial para el Análisis de Documentos Médicos en un Consultorio Escolar"**

**Por qué funciona:**
- Destaca la IA propia (sistema experto)
- Menciona documentos médicos (PDF/Word)
- Específico del contexto escolar
- Suena académico y profesional

---

### Opción 2 - Integradora/Multiplataforma
**"Implementación de una Plataforma Multicanal para la Gestión de Servicios Médicos Escolares con Sistema de IA para Diagnóstico Asistido"**

**Por qué funciona:**
- "Multicanal" = web (Angular) + móvil (Flutter)
- "Diagnóstico Asistido" = IA que ayuda a doctores
- Enfocado en servicios médicos
- Cubre todo el alcance del proyecto

---

### Opción 3 - Seguridad + IA
**"Sistema de Gestión Médica Escolar con Autenticación de Dos Factores e Inteligencia Artificial para el Procesamiento Inteligente de Documentos Clínicos"**

**Por qué funciona:**
- Destaca el 2FA (seguridad es importante)
- "Procesamiento inteligente" = IA
- Documentos clínicos suena profesional
- Balance entre seguridad e IA

---

### Opción 4 - Corta y Directa
**"Plataforma de Salud Escolar con IA: Sistema Experto para Análisis de Documentos Médicos y Gestión de Citas"**

**Por qué funciona:**
- Corta y fácil de recordar
- "Plataforma de Salud" es moderno
- Específica las 2 funciones principales (IA + citas)
- Ideal para presentaciones rápidas

---

### Opción 5 - Enfoque Académico/Institucional
**"Diseño e Implementación de un Sistema de Información Médica Escolar con Módulo de Inteligencia Artificial para la Validación de Diagnósticos"**

**Por qué funciona:**
- "Sistema de Información" = término académico estándar
- "Diseño e Implementación" = cubre todo el proceso
- "Validación de diagnósticos" = el doctor valida lo que sugiere la IA
- Muy formal para tesis/documentos académicos

---

## 🏆 Recomendación

**Para título de residencia, te recomiendo la Opción 2 o 3:**

- **Opción 2** si quieres destacar la integración multiplataforma (web + móvil)
- **Opción 3** si quieres destacar la seguridad (2FA) y la IA

**Para documentos formales (tesis, artículo):**
- **Opción 5** es la más académica

---

## 📌 Datos Técnicos para Justificar el Título

**Justificación de la IA implementada:**
- Sistema experto propio (no usa APIs externas)
- Procesamiento de lenguaje natural básico
- Reconocimiento de patrones médicos
- Base de conocimiento con reglas clínicas
- Confianza calculada para cada diagnóstico

**Justificación de la seguridad:**
- 2FA con códigos de 6 dígitos
- Rate limiting para prevenir ataques
- Auditoría de accesos (AuditLog)
- Roles diferenciados (RBAC)

**Justificación multicanal:**
- Backend Laravel REST API
- Frontend Angular (doctores/admins)
- App Flutter (alumnos/pacientes)
- Base de datos PostgreSQL en la nube (Neon)
